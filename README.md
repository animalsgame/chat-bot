# Появился установщик для быстрого запуска чат-бота!
Всё будет сделано автоматически, вам понадобится только токен бота.  
Проект находится по ссылке https://github.com/animalsgame/chat-bot-installer

# Установка
### 1: Скачайте архив с проектом, можно по прямой ссылке
https://github.com/animalsgame/chat-bot/archive/refs/heads/main.zip

Откройте архив, перенесите папку chat-bot-main из архива в любое удобное место, например на рабочий стол, перейдите в эту папку, там вы найдёте все файлы которые нужны для запуска бота!

### 2: Для запуска бота нужно установить nodejs (это javascript, но выполняется уже не в браузере)
#### Если у вас windows
nodejs уже находится в проекте, переходите к следующему шагу!
#### Если у вас linux (ubuntu, debian)
Откройте терминал и введите `apt-get install nodejs` подтвердите установку буквой Y

# Создание бота и получение токена
Если вы уже создали бота через игру, пропустите этот шаг.

Зайдите в игру, нажмите на ник системы (она мама всех ботов) `id -2` перейдите в информацию и нажмите на кнопку `создать бота` подтвердите создание, потом перейдите в управление, там будет кнопка для получения токена, скопируйте токен через правый клик, или клавишами `CTRL + C`

Если вы потеряли своего бота, вы можете вызвать его в чате или в привате `@bot1` где 1 = id бота

# Запуск
#### Токены теперь хранятся в файле `token.txt` один токен - одна строка.

Если у вас linux, вместо run файла нужно открыть терминал и ввести `node service.js` 

#### Для windows, чтобы быстрее открывать код бота в блокноте (есть файл `edit.bat`) после запуска нажмите enter.
Откройте текстовый файл в папке с ботом (`token.txt`) добавьте токен который вы получили у бота (можно вставить клавишами `CTRL + V`) Сохраните файл, быстрый способ это клавиши `CTRL + S`

Осталось запустить файл `run` если токен правильный, вы увидите в окне информацию о боте, его id, ник и тд. Смотрите свой приват в игре, бот должен сообщить что он в сети!

Бот работает! Уже можно играться готовыми командами, запомним что файл `bot.js` изменять нельзя, а вот `app.js` можно изменять (и нужно) но об этом чуть позже.

#### Код для ботов универсальный, если вы хотите запустить бота для другого проекта.
Получите токен бота через игру, откройте `token.txt` добавьте ещё одну строчку с токеном, сохраните файл и запустите файл `run`.

После изменения кода в `app.js` теперь изменения мгновенно применяются, перезапуск бота теперь не нужен!

# Минимальный пример app.js
Если нужен очень быстрый старт, или сломался код и нужно проверить работает ли бот, вы можете создать файл app.js с таким кодом,  
у бота будет только команда `пинг`
```js
module.exports = function(bot){

if(!bot.myBotInit){
bot.myBotInit=true;

// этот код выполняется один раз (когда бот запустится) чтобы каждый раз после изменения файла не отправлял уведомление что бот в сети, ведь он и так в сети!
console.log('бот', bot.botInfo);
bot.sendMessage(bot.botInfo.owner, 'Мой владелец, я в сети*em3*', {color:['#f1a0b3', '#FFFF00', '#54EFF7']});

}

bot.cmd('пинг',(user,words)=>{
if(words.length==0)return 'ПОНГ';
});

};
```

# Модули
Каждый модуль в коде содержит комментарии с примерами использования, все модули находятся в папке `modules`  
Важный модуль `botupdate.js` с помощью него вы можете через игру обновлять любой модуль,  
или даже клиент бота `bot.js` прямо через приват в игре, это очень удобно!  
Если в модуль вносятся изменения и публикуется код сюда, вы можете просто обновить модуль, не скачивая проект с ботом повторно.

### Команды доступны только владельцу бота
`bot update client` - обновляет клиент бота (файл bot.js)  
`bot update module имя` - обновляет модуль отдельно, вместо имя укажите имя модуля (например cats)

Ещё несколько команд  
`бот ник новый_ник` - устанавливает ник  
`бот цвет #FFFFFF` - устанавливает цвет ника на белый, цвета разделяются пробелом, если указано больше одного цвета, ник будет с градиентом!  
`бот пол ?` - устанавливает пол, по умолчанию ж, вместо ? укажите букву, м - мужской, ж - женский, н - нло  
`бот осебе текст` - устанавливает "о себе" в информации

#### Для подбора цвета вы можете использовать различные сервисы
Например https://colorscheme.ru там внизу справа будет код RGB, то что нужно для команды "бот цвет"

# Обновление июнь 2026
Теперь можно создавать кнопки, popup окна, ещё добавлен метод api для подарков!
#### Новый метод api `app.gifts`
Теперь можно получать список всех подарков, если вы работаете с подарками в коде, рекомендуется добавить код когда бот первый раз запускается (отправляет сообщение что бот в сети) а данные сохранятся в `bot.giftsList`
```js
bot.api('app.gifts',{},(res)=>{
if(res && Array.isArray(res))bot.giftsList = res;
});
```
А в нужных командах просто проверить, есть ли список подарков, если нет то отправить какое нибудь сообщение
```js
if(!bot.giftsList)return {msg:'Список подарков пуст'};
```
Метод возвращает масссив с объектами подарков, каждый объект содержит эти свойства
| свойство | описание |
| --- | --- |
| id | id |
| name | название |
| price | цена |

Примеры как работать с подарками
```js
// найти подарки по стоимости от 200 до 500 (включительно)
var gifts = bot.giftsList.filter(o=>o.price >= 200 && o.price <= 500);

// найти подарки по id
var ids = [1, 2, 3]; // какие подарки ищем
var gifts = bot.giftsList.filter(o=>ids.includes(o.id));
```

#### Кнопки и окна
Вся магия в свойстве `items` которое надо добавить к сообщению, а в текст сообщения добавить `{id}` где id это id предмета.  
Каждый предмет должен быть с уникальным id, по этому id можно обращаться!  
`items` это массив c объектами "предметов".  
Пример с кнопкой для отправки команды пинг
```js
{items:[{id:'button1', title:'тык', cmd:'пинг'}], msg:'Привет {button1}'}
```
Объект предмета может содержать эти свойства
| свойство | тип | описание | обязательно | доп. информация |
| --- | --- | --- | --- | --- |
| id | string | для обращения в тексте сообщения | да | должен быть уникальным |
| cmd | string | команда | да | после нажатия на кнопку будет отправлен текст из cmd (вместо ручного ввода) |
| title | string | название кнопки | нет | если не указано, будет стандартный текст для кнопки |
| color | string или array | цвет текста кнопки | нет | цвет должен быть в hex формате #FFFFFF, если color это массив, тогда текст кнопки будет с градиентом |
| tint | string | цвет, но для графики | нет | цвет должен быть в hex формате #FFFFFF, для создания эффекта с цветностью, для текста это не надо, а вот для цвета окон идеально |
| muted | boolean | тихая отправка | нет | текст из cmd отправится тихо (сообщения не будет видно) |
| payload | any (что угодно) | полезная нагрузка | нет | работает вместе с cmd, но если указан и payload то сообщение отправляется как со свойством muted (тихо) |
| giftid | number | id подарка | нет | если указан giftid тогда кнопка превращается в картинку с подарком, а если giftid равен 0, тогда вместо подарка будет картинка подарочной коробки |

#### Объект popup для создания окна (должен быть добавлен в объект предмета)
| свойство | тип | описание | обязательно | доп. информация |
| --- | --- | --- | --- | --- |
| text | string | содержимое окна, поддерживаются предметы и в окнах, но указывать items не надо, просто обращаться через {id} | да | - |
| title | string | название окна | нет | если не указано, будет стандартный текст |
| tint | string | цвет окна | нет | цвет должен быть в hex формате #FFFFFF |
| color | string или array | цвет текста кнопки | нет | цвет должен быть в hex формате #FFFFFF, если color это массив, тогда текст в окне будет с градиентом |

#### Объект media (должен быть добавлен в объект предмета)
| свойство | тип | описание | обязательно | доп. информация |
| --- | --- | --- | --- | --- |
| url | string | ссылка на файл | да | по ссылке будет загружен файл, если аудио то откроется плеер, загрузка возможна только с домена на котором находится игра |
| title | string | название файла | нет | если не указано, будет стандартный текст |
| ext | string | расширение файла | нет | если не указано, то расширение будет взято из url, иначе возьмётся из ext, это нужно чтобы понять какой формат файла из url (его может не быть в url) |

```js
// создание окна
{items:[{id:'popup1', title:'открыть окно', popup:{title:'Моё окно', tint:'#A2CFFC', text:'Содержимое окна'}}], msg:'Привет {popup1}'}

// создание медиа (аудио)
{items:[{id:'media1', title:'мелодия', media:{title:'Вот мелодия', url:'https://ag6.ru/musics/1.mp3', ext:'mp3'}}], msg:'Привет {media1}'}

// создание подарка с id 1
{items:[{id:'gift', giftid:1}], msg:'Вот подарок {gift}'}
```
### payload (полезная нагрузка)
В объект предмета можно добавить свойство `payload` и облегчить обработку команд без лишнего создания другой команды, payload работает со свойством `cmd` то есть если бот создаёт предмет с payload, нужно указать и свойство `cmd` (команда, куда придёт payload).  
Сам payload в обработчик приходит в виде объекта, проще всего достать его через свойство `value` и проверить через `typeof` что это строка.   
Пример обработки, создаём новую команду "тест", и добавляем payload к кнопке со значением secret, и получить доступ к секретному тексту можно только через кнопку!
```js
bot.cmd('тест',(user,words)=>{
var payload = (user.payload && typeof user.payload.value == 'string') ? user.payload.value : null;

if(payload == 'secret'){
return {msg:'Секретный текст'};
}

return {items:[{id:'button', title:'секрет', cmd:'тест', payload:'secret'}], msg:'Вот {button}'};
});
```

### свойство msgid для замены сообщения
Сообщение которое получает бот, содержит msgid, добраться до него можно через `user.msgid` в обработчике нужной команды.  
Чтобы заменить сообщение (а не отправлять как новое), нужно добавить это свойство при отправке сообщения.  
```js
{msgid:user.msgid, msg:'Привет'};
```
